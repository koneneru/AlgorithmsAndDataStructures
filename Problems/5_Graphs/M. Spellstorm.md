# Вихрь заклинаний
ограничение по времени на тест: 1 секунда  
ограничение по памяти на тест: 256 мегабайт  
ввод: стандартный ввод  
вывод: стандартный вывод

У игрового персонажа имеется несколько заклинаний, каждое из которых можно
мгновенно применить в бою для получения определенного преимущества. После
применения заклинание уходит на перезарядку и недоступно некоторое время.

Помимо непосредственного эффекта некоторые из заклинаний при применении также
могут *обновлять* какие-то из заклинаний — сбрасывать их перезарядку, позволяя
сразу же после этого применить обновленные заклинания вновь. Механика обновления
позволяет сочетать небольшое количество простых заклинаний в сложные и
разрушительные комбинации, которые при этом не требуют ожидания перезарядки ни
одного из заклинаний.

Ваша задача: для заданного набора заклинаний и правил их обновлений найти
наиболее эффективную и наиболее длинную последовательность их применений.

## Входные данные
В первой строке задается два целых числа $N$ и $M$
$(1 \leq N \leq 36; 0 \leq M \leq N^2)$ — количество заклинаний и количество
обновлений заклинаний.

Во второй строке задается строка из N различных символов — обозначения
заклинаний. Каждый из этих символов является цифрой или заглавной буквой
латинского алфавита.

В каждой из следующих M строк задано описание одного обновления заклинаний в
формате

*a resets b*

где *a* и *b* — обозначения заклинаний. При применении заклинания *a* заклинание
*b* обновляется и мгновенно становится доступным для использования.

Гарантируется, что ни одно из заклинаний не обновляет одно и то же заклинание
дважды.

## Выходные данные
В единственной строке без пробелов выведите наиболее длинную последовательность
заклинаний, все заклинания в которой могут быть мгновенно применены
последовательно.

Если подходящих ответов несколько, то выведите любой из них. Если
последовательность содержит больше 1000 заклинаний, то выведите вместо этого
«*spellstorm*». Кавычки выводить не требуется. Регистр каждой буквы имеет
значение.

## Примеры
**Входные данные**
``` no-hilights
5 4
AQWER
W resets A
E resets A
R resets A
R resets E
```
**Выходные данные**
``` no-hilights
QAEAWARAEA
```

**Входные данные**
``` no-hilights
7 3
QWER123
Q resets E
1 resets Q
E resets 1
```
**Выходные данные**
``` no-hilights
spellstorm
```

**Входные данные**
``` no-hilights
4 1
ABCD
A resets A
```
**Выходные данные**
``` no-hilights
spellstorm
```
