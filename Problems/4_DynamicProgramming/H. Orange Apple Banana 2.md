# Апельсинка, яблоко, банан \[2\]
ограничение по времени на тест: 1 секунда 
ограничение по памяти на тест: 256 мегабайт 
ввод: стандартный ввод 
вывод: стандартный вывод

Каждый день Даша любит начинать с свежего фрукта. Больше всего ей нравятся
апельсинки, яблоки и бананы. Сейчас у Даши осталось ***n*** фруктов и она хочет
распланировать свое меню на следующие ***n*** дней так, чтобы каждый день она
могла съедать по одному фрукту. Для того, чтобы выбрать действительно тот
вариант, который понравится ей больше всего, Даша решила рассмотреть их все.
Однако оказалось, что для совсем небольшого количества фруктов существует очень
много вариантов меню. Теперь Даша хочет посчитать их количество.

Даша считает два варианта меню различными, если в какой-то из ***n*** дней эти
варианты предлагают разные фрукты.

## Входные данные
В трех строках задаются по одному целому числу: ***o***, ***a*** и ***b***
— количество апельсинок, яблок и бананов у Даши соответственно.

Все числа лежат в диапазоне от **0** до **12** включительно. Гарантируется, что
у Даши есть хотя бы один фрукт.

## Выходные данные
Выведите одно целое число — количество различных вариантов меню, которое можно
составить из всех фруктов Даши.

## Примеры
**Входные данные**
```no-hilights
2
1
1
```
***Выходные данные***
```no-hilights
12
```
**Входные данные**
```no-hilights
4
4
4
```
***Выходные данные***
```no-hilights
34650
```
**Входные данные**
```no-hilights
0
1
0
```
***Выходные данные***
```no-hilights
1
```
**Входные данные**
```no-hilights
9
9
9
```
***Выходные данные***
```no-hilights
227873431500
```

## Примечание
Рассмотрим первый пример. Обозначим апельсинку за *O*, яблоко — за *A* и банан
— за *B*. Всего из **2** апельсинок, **1** яблока и **1** банана можно
составить **12** различных меню на **4** дня:

*BOOA*, *OBOA*, *OOBA*,

*BOAO*, *OBAO*, *OOAB*,

*BAOO*, *OABO*, *OAOB*,

*ABOO*, *AOBO*, *AOOB*.
