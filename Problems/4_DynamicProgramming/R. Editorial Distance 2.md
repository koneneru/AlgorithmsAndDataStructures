# Редакционное расстояние \[2\]
ограничение по времени на тест: 1 секунда  
ограничение по памяти на тест: 256 мегабайт  
ввод: стандартный ввод  
вывод: стандартный вывод

Редакционное расстояние между двумя строками — это минимальное количество
операций, которое потребуется для того, чтобы превратить одну строку в другую.
Каждая операция может быть одной из:
- удалить один символ на любой позиции;
- добавить один любой символ перед или после любого символа в строке;
- заменить один символ на любой позиции на любой другой символ.

Заданы две строки ***A*** и ***B***. Вам необходимо найти редакционное
расстояние между ***A*** и ***B*** или сообщить, что оно больше заданного
числа ***K***.

## Входные данные
В первой строке задана строка ***A***.

Во второй строке задана строка ***B***.

В третьей строке задано одно целое число ***K*** (**0** ⩽ ***K*** ⩽ **50**)
— лимит редакционного расстояния.

Строки ***A*** и ***B*** состоят только из символов с *ASCII* кодами от **33**
до **126** включительно, а их длины лежат в диапазоне от **1** до **10^5**
включительно.

## Выходные данные
Если искомое редакционное расстояние больше ***K***, то выведите **−1**. В ином
случае выведите редакционное расстояние.

## Примеры
**Входные данные**
```no-hilights
abacabadabacaba
abacabadabacaba
24
```
**Выходные данные**
```no-hilights
0
```
**Входные данные**
```no-hilights
abcdef
acdefu
7
```
**Выходные данные**
```no-hilights
2
```
**Входные данные**
```no-hilights
aaaaaaaaaa
bbbbbbbbbb
5
```
**Выходные данные**
```no-hilights
-1
```
